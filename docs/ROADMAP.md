# Roadmap / Vision

채용공고 자동 수집 봇의 장기 비전과 검토 중인 확장 아이디어를 정리합니다.
새 작업을 시작하기 전 이 문서를 참고해 큰 그림과 정합되는지 확인하세요.

## 최종 목표 파이프라인

사용자가 "지원 의사 결정 → 실제 지원 완료"까지 가는 마찰을 최소화하는 것이 목표입니다.

1. **공고 수집** — 원티드/사람인/잡코리아 (✅ 완료)
2. **AI 랭킹 + Discord 발송** — 이력서 기반 Top N 추천 (✅ 완료)
3. **선택 공고 → 맞춤 이력서 생성** — Discord 리액션으로 선택한 공고에 맞춰 이력서/자소서 자동 튜닝
4. **합격 시 면접 질문 생성** — 공고 + 맞춤 이력서 컨텍스트로 예상 질문 + 모범 답변 초안

## 단계별 설계 메모

### 3단계 — 맞춤 이력서 생성

- **트리거 후보**: Discord 리액션(`applied.json`의 `interested` 등) 또는 별도 CLI 명령
- **신규 use case**: `usecase/tailor_resume.py`
- **신규 어댑터**: `adapters/resume/claude_tailor.py` — `agent_rank.py`처럼 `claude -p` subprocess 패턴 재사용
- **입력**: `data/snapshots/{date}/{job_id}.md` + `resume/이력서.txt` + `resume/경력기술서.txt`
- **출력**: `resume/tailored/{job_id}.md`
- **운영 모드 결정 필요**: cron 1회 배치로 일괄 생성 vs `cli/bot.py` 상주 봇에 핸들러 추가 (실시간)

### 4단계 — 면접 질문 생성

- **트리거**: 별도 Discord 리액션 (예: ✅ = 서류 합격)
- **신규 use case**: `usecase/interview_prep.py`
- **신규 어댑터**: `adapters/interview/claude_questions.py`
- **컨텍스트**: 공고 원문 + 맞춤 이력서 둘 다 주입

## 검토 중인 확장 아이디어

채택 여부 미정. 우선순위·실효성 검증 후 단계적으로 도입.

### 파이프라인 보강
- **회사 리서치 단계** — 공고 선택 시 회사명으로 잡플래닛/블라인드/뉴스/깃허브 자동 수집 → 연봉·평판·기술스택 요약을 이력서 맞춤화 전에 컨텍스트로 주입
- **JD ↔ 이력서 갭 분석 (공고 단위)** — 맞춤 이력서 생성 *전에* "이 공고가 요구하는데 내 이력서엔 약한 항목" 리포트 (기존 `cli/resume_gap.py`를 공고 단위로 확장)
- **자소서 동시 생성** — 한국 채용 특성상 자소서 비중이 큼. 이력서 맞춤화와 같은 컨텍스트로 한 번에

### 4단계 면접 강화
- **회사별 과거 면접 후기 크롤링** (잡플래닛/블라인드) → 질문 생성 시 RAG 컨텍스트
- **모의 면접 모드** — 봇이 질문 → 사용자 답변 → Claude 피드백 (Discord thread 활용)
- **STAR 답변 템플릿 자동 생성** — 경력기술서 사실만으로 채운 초안

### 6단계 — 지원 후 트래킹
- `applied.json` 확장: 지원일/마감일/단계(서류·1차·최종) 기록
- 주간 "지원 현황 + 후속조치 알림" 발송
- 무응답 N일차 공고에 대한 후속 메일 초안 자동 생성

### 메타 학습 루프
- 사용자 ✅/❌ 패턴 분석 → AI 랭커 프롬프트 자동 보정 (예: "스타트업+원격 선호" 학습)
- 면접 합격/탈락 결과 입력 → 공고 유형별 전환율 통계

### 운영 측면
- **중복 공고 클러스터링** — 같은 포지션이 여러 사이트에 뜨면 1개로 묶어 발송
- **연봉 추정 모델** — 공고에 연봉이 없을 때 통계/과거 데이터로 추정 표시
- **대시보드** — Streamlit으로 풀 현황·매칭 점수 분포·지원 전환율 시각화

## 우선순위 추정

ROI 기준 상위 3개:
1. **회사 리서치** — 맞춤 이력서/면접 질문 품질을 동시에 끌어올림
2. **자소서 자동 생성** — 한국 채용 마찰 큰 부분
3. **지원 트래킹** — 6단계, 사용자 행동 데이터가 메타 학습 루프의 입력이 됨

## 아키텍처 원칙 (변경 없음)

확장 시에도 [CLAUDE.md](../CLAUDE.md)의 Clean Architecture 규칙을 유지:
- `domain/`, `usecase/`는 외부 의존 import 금지
- 외부 호출은 `adapters/`에 격리
- 새 use case는 포트만 의존, 진입점은 `cli/`에 추가
